نظرة عامة

منصة تبرعات تربط المتبرعين والجمعيات والمستفيدين. تم بناء باك إند كامل وربطه بقاعدة بيانات SQL Server مع توفير توثيق وتجربة مباشرة للواجهات.

الهدف: منصة تبرعات متكاملة
الدور: مطور باك إند
الناتج: واجهات API جاهزة للفرونت

التقنيات المستخدمة

ASP.NET Core Web API
Entity Framework Core
SQL Server
JWT Authentication
BCrypt لتشفير كلمات المرور
Swagger لتوثيق وتجربة الواجهات

هيكل المشروع

yad-elawn/ ├─ frontend/ ├─ backend/ │ └─ api/YadElAwn.Api/ ├─ ai/ ├─ database/ ├─ docs/

التوثيق والتجربة

تم نشر واجهة التوثيق على المسار الرئيسي لتسهيل التجربة.

http://localhost:5000/

الواجهات الأساسية

تسجيل الدخول
POST /api/auth/login
التسجيل حسب الدور
POST /api/registrations/donor POST /api/registrations/charity POST /api/registrations/beneficiary POST /api/registrations/admin
التبرعات
GET /api/donations GET /api/donations/{id} GET /api/donations/available POST /api/donations PATCH /api/donations/{id}/status
المواقع
GET /api/locations GET /api/locations/{id} POST /api/locations
الرسائل
GET /api/messages/inbox/{userId} POST /api/messages
الإشعارات
GET /api/notifications/{userId} POST /api/notifications
المطابقات
GET /api/matches POST /api/matches

الواجهات المحمية تتطلب إرسال التوكن في الهيدر.

Authorization: Bearer <TOKEN>

الأمان

  • تشفير كلمات المرور باستخدام BCrypt.
  • JWT لتأمين الوصول للواجهات المحمية.

التحديات والحلول

مشكلة عدم ظهور التوثيق
الحل:
  • تثبيت مسارات التوثيق وضبط الإعدادات.
  • تقديم واجهة التوثيق على المسار الرئيسي.
خطأ 500 في إنشاء التبرع
الحل:
  • حل مشكلة التكرار المرجعي في JSON.
  • اعتماد IgnoreCycles في إعدادات الإخراج.
اختلاف أسماء الجداول
الحل:
  • تثبيت أسماء الجداول يدويًا في إعدادات ORM.
modelBuilder.Entity<Donor>().ToTable("Donor")
