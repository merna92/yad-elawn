مين أنا عملت إيه؟ (مختصر واضح)

أنا اشتغلت على الباك إند لمشروع "يد العون". بنيت API كاملة وربطتها بقاعدة بيانات SQL Server، وعملت تسجيل مستخدمين بأدوار مختلفة، وتسجيل دخول بتوكن JWT للأمان. وكمان وفّرت واجهات للتبرعات والرسائل والإشعارات والمطابقة. التوثيق كله جاهز على Swagger عشان الفرونت يشتغل مباشرة.

التقنيات اللي استخدمتها

ASP.NET Core Web API
Entity Framework Core
SQL Server
JWT Authentication
BCrypt لتشفير كلمات المرور
Swagger لتوثيق وتجربة الواجهات

هيكل المشروع (منظم)

yad-elawn/ ├─ frontend/ ├─ backend/ │ └─ api/YadElAwn.Api/ ├─ ai/ ├─ database/ ├─ docs/

قسمت الريبو بوضوح عشان أي حد في الفريق يعرف يوصل بسرعة لكل جزء.

الواجهات المحمية تتطلب إرسال التوكن في الهيدر.

Authorization: Bearer <TOKEN>

الأمان

  • الباسورد يتخزن مشفر باستخدام BCrypt.
  • الـ JWT Token بيضمن إن أي طلب محمي لازم يكون من مستخدم مسجل دخول.

التحديات اللي قابلتني وحليتها

  • مشكلة ظهور Swagger: كانت مش بتظهر، أصلحت إعدادات المسار وخليتها تفتح على الصفحة الرئيسية.
  • خطأ 500 في التبرعات: بسبب دورة مرجعية في JSON، حليتها بـ IgnoreCycles.
  • اختلاف أسماء الجداول: ثبتّ أسماء الجداول يدويًا في EF Core عشان التطابق مع الداتابيز.

أسئلة متوقعة وإجاباتها (بصيغة ميرنا)

س: إزاي الفرونت يشتغل على الباك إند؟
ج: أولًا يعمل Register، بعد كده Login ياخد التوكن، وبعدين يبعت التوكن في أي طلب محمي.
س: ليه استخدمتي JWT؟
ج: عشان يبقى في أمان، ومافيش حد يقدر يستخدم الـ API من غير تسجيل دخول.
س: الباسورد بيتخزن إزاي؟
ج: بيتشفّر بـ BCrypt، يعني مش بيتخزن نص صريح في الداتابيز.
س: إيه الفرق بين Donor وCharity وBeneficiary؟
ج: كل دور ليه جدول وربط مختلف في الداتابيز، والتسجيل بيحدد الـ UserType بشكل واضح.
س: إيه اللي بيخلّي التبرع يظهر في متاح؟
ج: حالة التبرع Status، ولو كانت "Available" يظهر في فلتر التبرعات المتاحة.
س: التوثيق فين؟
ج: على Swagger في الصفحة الرئيسية: http://localhost:5000/
س: لو حصل خطأ 500 تاني، هتعملي إيه؟
ج: هراجع الـ Logs وأشوف هل المشكلة من الداتابيز ولا من الـ Model أو من Serialization، وأصلح السبب.
س: هل الباك إند جاهز للتسليم؟
ج: أيوه، اتعملت اختبارات لكل المسارات الأساسية واشتغلت بنجاح.
س: عملتي اختبار عملي؟
ج: أيوه، جرّبت التسجيل، الدخول، التبرعات، الرسائل، الإشعارات، والمطابقة وكلهم اشتغلوا.
