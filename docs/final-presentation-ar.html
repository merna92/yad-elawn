فكرة المشروع

يد العون منصة تبرعات تربط بين المتبرعين والجمعيات والمستفيدين.

تم بناء الباك إند باستخدام ASP.NET Core Web API وقاعدة بيانات SQL Server.

فكرة الداتابيز: تم تجهيز قاعدة البيانات أولًا، ثم تم ربط الباك إند بنفس الجداول والعلاقات باستخدام Entity Framework Core.

السويجر

Swagger UI: http://localhost:5000/

Swagger JSON: http://localhost:5000/swagger/v1/swagger.json

كيف تستخدمين Swagger:
  1. افتحي رابط السويجر.
  2. اختاري الـ endpoint الذي تريدين اختباره.
  3. اضغطي Try it out.
  4. املئي الـ request body.
  5. اضغطي Execute.
  6. اقرئي status code والـ response body.
ملاحظة الدخول: بعد نجاح تسجيل الدخول، انسخي الـ JWT token واستخدميه في الـ endpoints المحمية بهذه الصيغة: Authorization: Bearer <TOKEN>.

شرح الكود

استخدمي هذا الجزء كـ script كامل أثناء العرض.

1. تم بناء المشروع كموقع تبرعات باسم Yad ElAwn. 2. تم تجهيز قاعدة البيانات أولًا ثم تم ربط الباك إند بها باستخدام Entity Framework Core. 3. ملف Program.cs هو ملف التشغيل الرئيسي. فيه يتم تسجيل الخدمات، تفعيل Swagger، إعداد JWT Authentication، وربط SQL Server. 4. ملف ApplicationDbContext.cs يربط الكلاسات بجداول قاعدة البيانات ويحدد العلاقات بينها. 5. مجلد Models يحتوي على الكلاسات التي تمثل الجداول. 6. مجلد Dtos يحتوي على أشكال الطلبات والردود التي تستخدمها الـ API. 7. Controllers تستقبل HTTP requests وترسلها إلى الخدمات المناسبة. 8. Services تحتوي على منطق الشغل، مثل تسجيل الدخول، التسجيل، وإنشاء التبرعات. 9. Repositories تتعامل مع طبقة البيانات وتنفذ عمليات القراءة والكتابة. 10. JWT يستخدم حتى يحصل المستخدم على token بعد تسجيل الدخول ويستطيع الدخول إلى الـ endpoints المحمية. 11. BCrypt يستخدم لتشفير كلمات السر قبل تخزينها في قاعدة البيانات. 12. Transactions تُستخدم في التسجيل وإنشاء التبرعات حتى تنجح العملية كلها أو تفشل كلها معًا. 13. Swagger يفتح من http://localhost:5000/ ويستخدم لاختبار كل الـ APIs. 14. بعد تسجيل الدخول، يتم إرسال التوكن في الطلبات المحمية بهذا الشكل: Authorization: Bearer <TOKEN>. 15. أهم مسارات النظام هي: التسجيل، تسجيل الدخول، إنشاء التبرع، الرسائل، الإشعارات، والمطابقة. 16. دورة التنفيذ الكاملة هي: Frontend أو Swagger -> Controller -> Service -> Repository -> DbContext -> SQL Server -> Response.
